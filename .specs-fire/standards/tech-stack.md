# Tech Stack

## Overview

Office AI Agent is a cross-platform Electron desktop application. The main process runs Node.js (DB, LLM, IPC, orchestrator, skills); the renderer process runs React + Vite (UI). Communication between processes happens exclusively via Electron's IPC through a `contextBridge` preload (`window.officeAPI`).

## Core Technologies

### Language

**Primary**: TypeScript
**Version**: ^5.4.0 (strict mode enabled across all configs)

### Framework

**Framework**: Electron
**Version**: ^31.0.0
**Rationale**: Cross-platform desktop delivery with native OS integration and secure process isolation.

### Runtime

**Runtime**: Node.js (main process) + Chromium (renderer process)
**Version**: Bundled with Electron ^31

## Data Layer

### Database

**Type**: Embedded relational
**Database**: SQLite via `better-sqlite3`
**Version**: ^11.0.0

### ORM / Data Access

**Tool**: Direct `better-sqlite3` queries (no ORM)
**Rationale**: Synchronous API fits Electron main process; avoids ORM overhead for a local-first app.

Not configured.

## API Layer

### API Style

**Style**: Electron IPC (request/response via `ipcMain`/`ipcRenderer`). All handlers return `ApiResponse<T>`. Renderer calls via `window.officeAPI` exposed through `contextBridge`.

### Documentation

IPC channel names defined in `src/shared/types.ts`.
